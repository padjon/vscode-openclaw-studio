# Research Notes

## What exists now

- The official OpenClaw repository is large, active, and user-facing. As of March 14, 2026, `openclaw/openclaw` shows over 312k GitHub stars and positions the CLI onboarding wizard as the preferred install path.
- Official docs emphasize `openclaw onboard`, `openclaw doctor`, gateway config, plugin manifests, and strict validation for `openclaw.plugin.json`.
- The plugin manifest docs show a concrete opportunity for IDE tooling: plugin manifests are mandatory and invalid manifests block config validation.

## Product implications

- A successful extension should complement the CLI instead of replacing it.
- The biggest IDE-native wins are config safety, plugin authoring, and one-click launch flows.
- Sponsorship should feel earned through saved time, not gated features.
